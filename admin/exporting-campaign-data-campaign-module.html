ビューイベントは、キャンペーン対応プレーヤーを使用してビデオが視聴されるときに記録されます。ユーザーがキャンペーンのリードフォームにデータを入力すると、リードデータがキャプチャされます。キャンペーンモジュールには、次の2種類のリードフォームがあります。

  • キャンペーンのリードフォーム- これらのフォームは、Campaign モジュールの一連の定義済みフィールドから選択して作成されます。これらのフォームは、リードデータを Video Cloud に保存します。キャンペーン リード フォームの作成については、次を参照してください。キャンペーン リード フォームの作成.
  • カスタム リードフォーム- これらのフォームは、マーケティング オートメーション プラットフォームを使用して構築されています。その後、フォーム埋め込みコードがキャンペーンモジュールで使用され、カスタムリードフォームが作成されます。これらのフォームは、リードデータをマーケティングオートメーションプラットフォームに送信します。リードフォームの作成については、マーケティングプラットフォーム用のドキュメントを参照してください。
    • Eloqua
    • Marketo
    • セールスフォース
    • HubSpot

キャンペーンモジュールからリードデータをエクスポートする場合、キャンペーンリードフォームを使用してキャプチャしたデータだけがエクスポートされます。カスタムリードフォームからのリードデータは、マーケティング自動化ソフトウェアに保存されます。

データをエクスポートするには、次の手順を実行します。

  1. 開く運動モジュール。
  2. クリックキャンペーン データのエクスポート左のナビゲーションにあります。
    注:[データのエクスポート] メニューにアクセスするには、アカウント管理者である必要があります。
  3. を選択データの接続からデータをエクスポートします。のデータの接続ドロップダウンには、構成されているすべての統合が一覧表示されます。
  4. エクスポートするデータを選択します。
    • イベントを見る- キャンペーン対応プレイヤーによる記録
    • リード- リードフォームを使用してキャプチャ
  5. 日付ピッカーを使用して日付を選択します開始日と終了日.
  6. を選択エクスポート形式.
    • カンマ区切り（CSV）
    • Excel（XLS）
  7. クリック書き出すリードデータをダウンロードします。
  8. リードフォームとビューイベントデータは、別々のファイルにダウンロードされます。 リードdata には、リードフォームに入力されたデータと次のフィールドが含まれます。
    • 外部 ID - これは、[最近のすべてのアクティビティ] ページに表示される外部 ID です。Oracle Eloqua の場合は、Eloqua GUID です。マルケトでは、それはマルケートのGUIDです
    • プレーヤーID・フォーム表示時に使用していたプレイヤーのプレイヤーID
    • ビデオID - フォームが表示されたときに再生されていたビデオのビデオ ID
    • ページの URL - リード フォームを表示したページの URL (ギャラリー エクスペリエンスのみ)
    • 作成日時- データがキャプチャされた日時
    • 経験- データを収集したギャラリー エクスペリエンス (ギャラリー エクスペリエンスのみ)

    イベントを見るデータには次のフィールドが含まれます。

    • 外部 ID - これは、[最近のすべてのアクティビティ] ページに表示される外部 ID です。Oracle Eloqua の場合は、Eloqua GUID です。マルケトでは、それはマルケートのGUIDです
    • ビデオID - 動画の動画 ID
    • ビデオ名- ビデオの名前
    • 見た- 視聴されたビデオのパーセンテージ (最も近いパーセンテージまで)
    • 視聴時間- 総視聴時間
    • プレーヤーID - 動画の視聴に使用したプレーヤーのプレーヤー ID
    • ページの URL - プレーヤーを含むページの URL
    • 作成日時- データがキャプチャされた日時
    • 更新日時- イベントが最後に保存された日付と時刻。これは、視聴者が視聴を停止した時刻に大まかに変換されます。ビデオの長さが 1 分だけの場合、createdAt と updatedAt は非常に近い値になります。
    • isSynced - このビュー イベントがパブリッシャーのマーケティング オートメーション ツールに自動的にエクスポートされたかどうかを指定します
    • トラッキングID - URL パラメータとして渡されるトラッキング ID。トラッキング ID の操作については、を参照してください。 Campaign モジュールとのカスタム統合の作成.
    • UTM フィールド- utmSource、utmMedium、utmCampaign、utmTerm、および utmContent。UTM フィールドの操作については、次を参照してください。 Campaign モジュールでの UTM フィールドの操作.
